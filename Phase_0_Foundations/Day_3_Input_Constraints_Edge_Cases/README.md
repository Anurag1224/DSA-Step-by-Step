# 🟢 Day 3: Input Constraints & Edge Cases

**Stop writing code that fails on rare inputs. Think defensively.**

---

## Why Edge Cases Matter

Most bugs happen at boundaries. Input constraints define what's possible. Ignoring them = runtime errors.

---

## Learning Objectives

- Handle empty/null inputs gracefully
- Understand constraint limits (N=1, N=10^5, etc.)
- Prevent division by zero
- Check array bounds
- Handle negative numbers correctly
- Validate string lengths

---

## Problems to Solve

### 1. Safe Array Minimum
Find minimum in array, handle empty array case.

### 2. String Length Validator
Check string meets length constraints.

### 3. Division with Zero Check
Safe division function with error handling.

### 4. Boundary Sum
Sum array elements, handle overflow potential.

### 5. Negative Number Handler
Process numbers that could be negative.

---

## Key Takeaways

- Always check input constraints first
- Edge cases are not "corner cases" — they're expected
- Defensive programming prevents crashes
- Constraints guide your algorithm choice
