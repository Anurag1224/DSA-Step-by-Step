# 🟢 PHASE 0: Foundations

**The Hidden Layer That Makes Everything Possible**

---

## Why PHASE 0 Matters

Most people skip foundations. They jump to arrays and trees. Then they struggle because:

- They don't understand complexity
- They make overflow mistakes
- They forget edge cases
- They write fragile code

This phase prevents all of that.

---

## What You'll Learn in PHASE 0

### Day 1: Time & Space Complexity
Learn to measure algorithm efficiency without running it. Understand big-O intuitively.

### Day 2: Modulo, Overflow & Fast Power
Prevent silent data corruption. Learn mathematical foundations used everywhere.

### Day 3: Input Constraints & Edge Cases
Stop writing code that fails on rare inputs. Think defensively.

### Day 4: Recursion Stack Understanding
Why deep recursion crashes. How stack memory works.

### Day 5: Math Fundamentals (GCD, LCM, Primes)
Mathematical tools that appear in every phase.

### Day 6: Bitwise Basics
Efficient manipulation. Pattern recognition at bit level.

### Day 7: Dry Run & Problem-Solving Mindset
Think before coding. Verify before submitting.

---

## Philosophy for PHASE 0

> "You can't optimize what you don't understand."
> "Edge cases aren't exceptions — they're the rule."
> "Clean code is not optional."

---

## How to Approach PHASE 0

1. **Read completely** — don't skip sections
2. **Dry run every example** — trace it yourself
3. **Code along** — don't just read code
4. **Test edge cases** — explicitly
5. **Commit after each day** — meaningful commits
