# 🔄 Day 4: Recursion Stack Understanding

**Why Deep Recursion Crashes & How Stack Memory Works**

---

## Today's Focus

Recursion is elegant but dangerous. Today you'll learn:

- ✅ How recursion uses stack memory
- ✅ Why deep recursion causes stack overflow
- ✅ How to calculate recursion depth safely
- ✅ When to prefer iteration over recursion

---

## Problems to Solve

### 1. Factorial with Recursion
Calculate factorial using recursion. Observe stack usage.

### 2. Fibonacci with Recursion
Compute Fibonacci numbers recursively. See exponential explosion.

### 3. Tower of Hanoi
Solve Tower of Hanoi with recursion. Understand call stack depth.

### 4. Safe Recursion Depth
Implement recursion with depth checking to prevent crashes.

---

## Key Concepts

- **Stack Frame**: Memory allocated for each function call
- **Stack Overflow**: When recursion depth exceeds stack limit
- **Tail Recursion**: When compiler can optimize stack usage
- **Iteration vs Recursion**: When to choose each approach

---

## Learning Goals

By the end of today, you'll:
- Understand why recursion can crash programs
- Know how to estimate safe recursion depth
- Be able to convert simple recursive functions to iterative
- Recognize when recursion is appropriate vs iteration
