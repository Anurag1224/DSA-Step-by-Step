## 💡 Learning Tips

### Before Each Coding Session
```
1. Read the README completely
2. Don't look at solution yet
3. Think for 10-15 minutes
4. Write your own code
5. Then compare with solution
6. Learn what you missed
```

### Dry Run Discipline
- Always trace examples on paper first
- Verify your logic before coding
- This catches 80% of bugs early

### Edge Case Mindset
- Ask: "What if n = 0?"
- Ask: "What if input is negative?"
- Ask: "What if input is huge?"
- Ask: "What breaks if this edge case happens?"

### Commit Frequently
- After each day completion
- With meaningful messages
- Git history becomes your learning portfolio
